## Directory Purposes

**`cmd/server/`:**
- Purpose: Single Go package вЂ” the entire application
- Contains: All `.go` source files (8 files), plus embedded templates/assets (15+ files in `templates/`)
- Key files: `main.go` (entrypoint + routes), `models.go` (types), `db.go` (data access), `auth.go` (auth/sessions), `handlers.go` (page + API handlers), `api_handlers.go` (JSON API handlers), `utils.go` (helpers), `main_test.go` (tests)

**`cmd/server/templates/`:**
- Purpose: Embedded static assets compiled into the binary
- Contains: Go html templates (`.html`), CSS (`.css`), JavaScript (`.js`), vendored HTMX (`htmx.min.js`)
- Important: All HTML template files use `{{define "name"}}` blocks for named templates
- Components directory contains reusable partials included by page templates

**`cmd/server/templates/components/`:**
- Purpose: Reusable template partials imported by page templates
- Contains: `head.html`, `nav.html`, `activities_table.html`, `activity_modal.html`, `users_section.html`, `user_row.html`, `user_edit_row.html`
- Usage: Included via `{{template "component_name" .}}` from page templates

**`tmp/`:**
- Purpose: Reference copy of the old application architecture (Bun/TypeScript/Hono version). DO NOT MODIFY.
- Contains: TypeScript source, old server, Drizzle config, Docker Compose, etc.
- Committed: Yes вЂ” kept as reference for migration comparison

**`bin/` and `.tmp/`:**
- Purpose: Build artifacts
- Generated: Yes (go build, Air)
- Committed: No (gitignored)

## Key File Locations

**Entry Points:**
- `cmd/server/main.go`: `main()` function + `routes()` method + template parsing + `//go:embed` directive

**Configuration:**
- `cmd/server/utils.go` (lines 17-80): `loadConfig()` вЂ” reads all env vars into `Config` struct
- `cmd/server/utils.go` (lines 89-122): `loadDotEnv()` вЂ” custom .env file parser
- `.env.example`: Template for required environment variables
- `.air.toml`: Hot-reload configuration for development

**Core Logic:**
- `cmd/server/handlers.go`: All HTML page handlers and primary API handlers (login, dashboard, activities, admin, printing, product lookup)
- `cmd/server/api_handlers.go`: Dedicated JSON API handlers (dashboard activities, admin CRUD via JSON)
- `cmd/server/auth.go`: Session token creation/validation, CSRF protection, role-based access control
- `cmd/server/db.go`: Postgres migrations, CRUD operations, Oracle queries with read-only guard
- `cmd/server/models.go`: All struct definitions (`App`, `Config`, `User`, `Activity`, `ProductVerification`, `OracleProduct`, etc.)
- `cmd/server/utils.go`: Utility functions (config, logging middleware, rate limiter, security headers, `writeJSON`, `parseFilters`)

**Testing:**
- `cmd/server/main_test.go`: All test cases. Single file, no external test framework. Tests: template parsing, SQL read-only guard, .env loading, role validation, SQL comment removal, random strings, token creation, rate limiter, security headers, CSRF middleware, config loading

## Naming Conventions

**Files:**
- Go source files: `snake_case.go` вЂ” e.g., `main.go`, `handlers.go`, `api_handlers.go`, `main_test.go`
- Template files: `snake_case.html` вЂ” e.g., `login.html`, `dashboard.html`, `activity_modal.html`
- CSS files: `snake_case.css` вЂ” e.g., `style.css`, `admin.css`
- JS files: `snake_case.js` вЂ” e.g., `app.js`, `dashboard.js`, `admin.js`
- Config files: `dot.prefix` вЂ” `.env.example`, `.air.toml`, `.gitignore`

**Functions (Go):**
- Exported: `PascalCase` вЂ” e.g., `QueryContext`, `QueryRowContext` (on `OracleReader`)
- Unexported: `camelCase` вЂ” e.g., `loadConfig`, `loadDotEnv`, `makeToken`, `currentUser`, `listActivities`, `writeJSON`, `parseFilters`
- Method receivers: Lowercase with receiver type вЂ” e.g., `func (a *App) render()`, `func (rl *rateLimiter) allow()`
- Test functions: `TestCamelCase` вЂ” e.g., `TestTemplatesParse`, `TestOracleReadOnlySQLGuard`, `TestHealthCheckResponse`

**Variables:**
- Go: `camelCase` вЂ” e.g., `cfg`, `pg`, `ora`, `tpl`, `mux`, `u`, `act`, `items`
- Constants (CSS in Go): Uppercase const block not used вЂ” CSS is a plain string const `styleCSS` and `adminCSS` embedded in `main.go`

**Types/Structs:**
- Domain models: `PascalCase` вЂ” `User`, `UserRow`, `Activity`, `ActivityFilters`, `FilterOptions`, `ProductVerification`, `OracleProduct`, `OracleEmpresa`, `OracleLocal`, `Config`, `App`
- API DTOs: `APIPrefixed` вЂ” `APIActivity`, `APIProductVerification`, `APIUser`
- Internal: `finalizeReq`, `rateEntry`, `rateLimiter`, `logWriter`, `Bundle`

## Where to Add New Code

**New Feature (e.g., new page):**
- Primary handler: Add a new method on `*App` in `cmd/server/handlers.go` (for HTML pages) or `cmd/server/api_handlers.go` (for JSON API endpoints)
- Route registration: Add `mux.HandleFunc(...)` call in `routes()` method in `cmd/server/main.go`
- Template: Create new `.html` file in `cmd/server/templates/` with `{{define "name"}}` block
- Data access: Add new query method in `cmd/server/db.go`
- Test: Add test function to `cmd/server/main_test.go`

**New Component/Module (e.g., a new reusable template partial):**
- Create `.html` file in `cmd/server/templates/components/` with `{{define "component_name"}}`
- Include via `{{template "component_name" .}}` from page templates

**New Middleware:**
- Add to `cmd/server/auth.go` (auth-related) or `cmd/server/utils.go` (generic)
- Chain in `main()` at `cmd/server/main.go:63`

**New Data Model:**
- Add struct to `cmd/server/models.go` (or `cmd/server/api_handlers.go` for API-specific DTOs)
- Add table migration in `migrate()` in `cmd/server/db.go`
- Add CRUD methods in `cmd/server/db.go`

**Utilities:**
- Shared helpers: `cmd/server/utils.go`

**Tests:**
- All tests: `cmd/server/main_test.go` (single file, no external framework)
- Naming: `TestCamelCase`
