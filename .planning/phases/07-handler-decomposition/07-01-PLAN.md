```go
func (a *App) apiFinalizar(w http.ResponseWriter, r *http.Request, u *User) {
    var req finalizeReq
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        a.handleError(w, r, &AppError{Code: ErrCodeBadRequest, Message: "JSON inválido", HTTPStatus: http.StatusBadRequest})
        return
    }
    if req.Empresa == 0 || req.Rua == "" || req.SeqLocal == 0 {
        a.handleError(w, r, &AppError{Code: ErrCodeBadRequest, Message: "Campos obrigatórios ausentes", HTTPStatus: http.StatusBadRequest})
        return
    }
    result, err := a.finalizeActivity(r.Context(), req, u.ID)
    if err != nil {
        a.handleError(w, r, err)
        return
    }
    writeJSON(w, http.StatusOK, map[string]any{
        "success": true, "atividadeId": result.ActivityID,
        "dataFim": result.DataFim, "divergences": result.Divergences,
        "ruptures": result.Ruptures, "replenishments": result.Replenishments,
    })
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP handler → service | Request data crosses from HTTP layer to service layer; service must not trust that handler validated correctly |
| Service → DB | SQL queries execute against Postgres; parameterized queries preserved during extraction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Elevation of Privilege | `finalizeActivity` service | mitigate | Permission check is already handled by middleware (`requireAPIRole`) before handler runs — service trusts caller's userID param. Service does NOT re-check user role since `apiFinalizar` is role-gated at the route level. |
| T-07-02 | Tampering | SQL in service function | mitigate | All pgx queries use parameterized `$1, $2, $3` placeholders — existing pattern preserved identically during extraction. No string-concatenated SQL in the extracted code. |
| T-07-03 | Information Disclosure | Error messages in service returns | mitigate | Service returns `*AppError` with same Portuguese messages as original handler. handler's `handleError` already controls which details reach the HTTP response (no stack traces, no internal state leaked). |
| T-07-04 | Repudiation | Transaction boundary leak | mitigate | `finalizeActivity` fully owns BeginTx/defer Rollback/Commit. Handler never touches `a.pg` or `tx`. If Commit fails, defer Rollback fires — same safety as original. |
| T-07-05 | Information Disclosure | Empty slice vs nil in response | mitigate | `FinalizarResult.Divergences/Ruptures/Replenishments` initialized with `make([]map[string]any, 0)` (not nil) — guarantees JSON `[]` not `null`, matching original behavior. |
</threat_model>
