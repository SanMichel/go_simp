   - `&AppError{Code: ErrCodeBadRequest, Message: "Role inválido", HTTPStatus: http.StatusBadRequest}`

3. **Target user lookup** — `a.findUserByID(ctx, targetID)`. If not found, return error:
   - `&AppError{Code: ErrCodeNotFound, Message: "Usuário não encontrado", HTTPStatus: http.StatusNotFound, Err: err}`

5. **Conditional DB update** — same as existing:
   - If password != "": bcrypt hash via `bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)`, then `a.pg.ExecContext(ctx, "UPDATE users SET role=$1, password=$2, last_token_at=now() WHERE id=$3", role, string(hash), targetID)`

```go
func (a *App) adminUpdateUser(w http.ResponseWriter, r *http.Request) {
    currentUser := r.Context().Value(ctxUser).(*User)
    id, _ := strconv.Atoi(r.PathValue("id"))
    if err := r.ParseForm(); err != nil {
        a.handleError(w, r, &AppError{Code: ErrCodeBadRequest, Message: "Erro ao processar formulário", HTTPStatus: http.StatusBadRequest, Err: err})
        return
    }
    role := r.FormValue("role")
    password := r.FormValue("password")
    u, err := a.updateUserAdmin(r.Context(), currentUser.ID, id, role, password)
    if err != nil {
        a.handleError(w, r, err)
        return
    }
    a.render(w, "user_row", map[string]any{"RowUser": u})
}
```

The handler keeps: extracting currentUser from context, parsing form values, calling service, handling error, rendering response. The self-edit guard and sysadmin protection are now in the service.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP handler → shared service | adminUpdateUser (HTML form) and apiAdminUserUpdate (JSON API) both cross into the same `updateUserAdmin` service — service must treat both callers identically for permission checks |
| Service → DB | UPDATE queries execute against users table; parameterized `$1,$2,$3` preserved |
| BulkDetails loop → activityDetailsData | Service delegates to existing `a.activityDetailsData` — same Oracle/Postgres access pattern, same security boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Elevation of Privilege | `updateUserAdmin` service | mitigate | Service re-implements identical permission checks from original handlers: self-edit guard (targetID != currentUserID), sysadmin protection (non-sysadmin can't edit sysadmin). Both handlers are already route-gated by `requireRole("sysadmin")` / `requireAPIRole("sysadmin")`. Service adds defense-in-depth. |
| T-07-07 | Tampering | SQL injection via role/password params | mitigate | All DB queries use pgx parameterized placeholders (`$1`, `$2`, `$3`). Same pattern as original handlers — no string concatenation. |
| T-07-08 | Information Disclosure | Error message mismatch between HTML and JSON handlers | mitigate | Both handlers call the same `handleError` dispatcher which controls response format. Service returns consistent `*AppError` values regardless of caller. |
| T-07-09 | Information Disclosure | apiDashboardBulkDetails error handling change | accept | When `a.activityDetailsData` returns error for an ID, `bulkActivityDetails` silently skips that ID (matching original handler behavior). Error details are NOT exposed to client — same safety as current. |
| T-07-10 | Information Disclosure | Empty slice vs nil in BulkDetails response | mitigate | `bulkActivityDetails` initializes bundles as `var bundles []FlatBundle` and has nil-to-empty-slice guard (`if bundles == nil { bundles = []FlatBundle{} }`) — preserves original behavior that produces JSON `[]` not `null`. |
| T-07-11 | Elevation of Privilege | Role validation bypass | mitigate | `validRole(role)` check is preserved in `updateUserAdmin` service. Both existing test paths exercise this. If removed, `TestUpdateUserAdmin_Success` would fail (invalid role rejection). |
</threat_model>
