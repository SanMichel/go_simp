<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → requireAtividadesRole middleware | Unauthenticated requests cross auth boundary |
| HTTP request → atividadesLoginPage handler | Login page template rendering |
| go:embed → templates/atividades/* | File system boundary for embedded assets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Spoofing | requireAtividadesRole redirect URL | mitigate | Middleware redirects to `/atividades/login` for unauthenticated requests; explicit path (not user-controlled) |
| T-08-06 | Information Disclosure | Handler renders template before auth check | mitigate | atividadesLoginPage checks auth FIRST (currentUser), then renders template. Authenticated users are redirected, not served the login page. |
| T-08-07 | Denial of Service | go:embed or template parse failure | mitigate | parseTemplates() uses template.Must — panics on startup if embed/ParseFS patterns are wrong, preventing a half-broken server from starting |
| T-08-08 | Elevation of Privilege | requireAtividadesRole vs requireRole | mitigate | requireAtividadesRole uses identical role-checking logic to requireRole — only the redirect URL differs. No role bypass possible. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced in this phase. All changes are in existing Go files with stdlib dependencies only. |
</threat_model>
