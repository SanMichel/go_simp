       - `sql.Open("pgx", url)` + `PingContext(context.Background())`

       - Calls `testDB(t)` to get Postgres connection

       - Direct HMAC token construction (duplicates makeToken logic without DB dependency)
       - `json.Marshal{ID, Exp: now+SessionTTL, Iat: now}` ‚Üí `base64.RawURLEncoding` ‚Üí `hmac-sha256(app.cfg.SessionSecret)` ‚Üí `body + "." + sig`

    4. `testUser(t *testing.T, app *App, username, role, password string) *User`:
       - `bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)`
       - `INSERT INTO users (username, password, role) VALUES ($1,$2,$3) RETURNING id, username, password, role, last_token_at`

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-06-02 | Information Disclosure | handleError wrapping | mitigate | handleError wraps non-AppError with `ErrCodeInternal` + generic "Erro interno do servidor" ‚Äî raw error text never reaches client. TestHandleErrorNonAppError asserts this |
| T-06-03 | Elevation of Privilege | Test user with known password | mitigate | cleanupTestData deletes `users WHERE username LIKE 'test_%'` in t.Cleanup(). No test user leaks to production DB |
| T-06-04 | Spoofing | Coverage masking auth gaps | accept | Coverage reports show which auth paths are tested. Manual review of uncovered lines in auth.go before phase sign-off |
| T-06-05 | Repudiation | Test helper with test secret | accept | Test secret `"test-secret-32-chars-minimum-length!"` is 32+ chars but weak. Only used in test environments where AppEnv="test" |
| T-06-SC | Tampering | Go stdlib testing packages | mitigate | No new external packages introduced. All testing uses Go standard library. verify with `go mod verify` |
</threat_model>
