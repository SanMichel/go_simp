<objective>
Create the error handling foundation: AppError type, handleError dispatcher, Validator type, fixed writeJSON, panic recovery middleware, request ID middleware, and slog structured logging.

Purpose: Replace ad-hoc error formatting with a centralized error handling pipeline. Every error flows through handleError() which logs, routes to HTML/JSON/text, and returns consistent responses. All handler code after this plan uses `a.handleError(w, r, err)` instead of inline `writeJSON(...)` or `http.Error(...)`.

Output: errors.go, validation.go, fixed utils.go (writeJSON + recovery + slog log middleware), slog init in main.go, error_toast.html template.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client→API | Untrusted HTTP requests arrive at /api/* endpoints |
| Client→Page | Untrusted form submissions arrive at page handlers |
| Handler→DB | Internal — AppError wraps DB errors; raw errors never reach client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | handleError dispatcher | mitigate | AppError.Message is always user-safe Portuguese; raw/wrapped error (Err) is logged via slog but never included in response body |
| T-05-02 | Denial of Service | recoveryMiddleware | mitigate | recoveryMiddleware wraps outermost handler; catches all panics via defer/recover, logs stack trace via debug.Stack() to stderr, returns HTTP 500 |
| T-05-03 | Tampering | writeJSON | mitigate | Buffer-first encoding: encode JSON to bytes.Buffer before writing WriteHeader; if encoding fails, write 500 instead of 200 with partial body |
| T-05-04 | Information Disclosure | handleError non-AppError wrapping | mitigate | Any error passed to handleError that is not an AppError is wrapped as ErrCodeInternal with generic "Erro interno do servidor" message; raw error text never reaches client |
| T-05-05 | Spoofing | requestIDMiddleware | accept | Request ID is generated locally (crypto/rand, 64-bit hex); no cryptographic validation needed for tracing |
</threat_model>
