<objective>
Migrate all inline error formatting in api_handlers.go to use the centralized a.handleError() dispatcher with AppError types, and replace log.Printf with structured slog calls.

Purpose: API handlers (admin CRUD, dashboard data) deliver errors through the same standardized pipeline as page handlers — structured logging, consistent JSON error shape `{"error": msg, "code": code}`, and centralized dispatch.

Output: api_handlers.go with all 12+ error paths converted from `writeJSON(w, code, map[string]string{"error":...})` to `a.handleError(w, r, &AppError{...})`, and all `log.Printf("error:...")` to `slog.ErrorContext`/`slog.WarnContext`.
</objective>
